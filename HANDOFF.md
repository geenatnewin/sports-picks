## What this is

A sports betting picks web app covering the **2026 World Cup**. It pulls real sportsbook odds (moneyline/spread/totals + player props), prediction-market probabilities (Kalshi + Polymarket), team stats, and recent form, feeds it all to Claude, and shows **two** AI picks per match, ranked purely by how likely each is to actually hit (Moneyline/Tie, Spread, Totals, and player props are all eligible candidates — no separate "value" pick, and the AI is explicitly told not to just mechanically grab the two shortest-odds favorites). Each pick has its own explanation and an optional "counterpoint" — only shown when there's a genuine, credible reason it might not hit, omitted entirely for near-locks (kept deliberately, confirmed cheap to keep — see Session 7). There's also a right-side **"AI Parlay" box** (Session 10): the same AI call also builds one best 3- or 4-leg parlay for the day across all matches, shown with combined odds and an "Add All to Parlay Slip" button. Plus a manual Game/Player Props browser you can tap to build a parlay, track placed slips, and a floating "My Picks" panel. Golf support was fully removed in Session 5; **MLB support was added in Session 7 and fully removed again in Session 9** (see below) — both were built, shipped, then deleted once no longer wanted.

Branding: displayed name is **"Dylan Harper's 'Trust Me' Locks"**. The codebase/repo/folder are still named `sports-picks` — only the Vercel project itself was renamed to `dylanharperpicks`.

### Squad value data-quality notes (Session 27)

`current_national_team_id` in the transfermarkt-datasets `players.csv` tags every player in a country's broader senior-team-eligible pool, **not** just a 26-man tournament roster — confirmed directly (Morocco alone has 82 tagged players). Summing the full pool overstated real squad value by 25-65% against Transfermarkt's own published totals for the teams where that ground truth exists (Argentina, Brazil, Morocco, USA — `national_teams.csv`'s own `total_market_value` column, which is separately null for several major teams including France/England/Spain/Portugal). Fixed by capping to the top-26-by-value players per team, which lands much closer (~20-35% over ground truth) but is still an approximation, not an exact roster — there's no free source for actual named 26-man squads, so this is the best available without scraping. Labelled honestly in code comments as an approximation, per user's explicit choice this session.

Also found and fixed two real team-name mismatches between The Odds API (used everywhere else in this app) and this dataset: `"USA"` vs `"United States"`, and `"South Korea"` vs `"Korea, South"` — small alias maps added in both new lib files. Every other 2026 World Cup team name checked matched directly with no alias needed. Some real 2026 teams (Ivory Coast, Cape Verde confirmed) simply don't appear in this dataset's `national_teams.csv` at all — handled gracefully (returns `null`, line omitted from the prompt), not a bug.
